# Contribuyendo a Investigación sobre la eficiencia en la generación automática de código utilizando modelos de lenguaje masivos.

## Cómo contribuir
Estamos encantados de recibir contribuciones de la comunidad. Aquí hay algunas pautas que te ayudarán a hacerlo de manera efectiva.

### Reportando errores
- Utiliza el sistema de issues de GitHub para reportar errores.
- Describe el error que has encontrado y cómo puede ser replicado.

### Sugerencias
- Si tienes sugerencias para mejorar el código, por favor discútelas en los issues antes de trabajar en ellos.

### Proceso de pull request
- Asegúrate de que tu código sigue el estilo del proyecto.
- Escribe [tests](#) para tu nuevo código.
- Mantén tus pull requests lo más pequeños posible, idealmente deberían abordar un solo tema.

### Pruebas
- Proporciona detalles sobre cómo probar las funcionalidades que has implementado o modificado.

Gracias por tu contribución!
